
# 🚨 CRITICAL FIX FOR 502 ERROR 🚨

The issue is a **Port Mismatch**.
Your server (Caddy/Nginx) expects the backend to be on **Port 9101**.
The current stack is putting it on **Port 8080**.

Because of this, the "Door is Closed" and you see a 502 Error.

## 1. Copy This EXACT YAML into Portainer Stack Editor

Replace the entire content with this:

```yaml
services:
  postgres:
    image: postgres:15-alpine
    container_name: vending_db
    restart: unless-stopped
    environment:
      POSTGRES_DB: vending
      POSTGRES_USER: vending_user
      POSTGRES_PASSWORD: iq200196Qr51
    volumes:
      - postgres_data:/var/lib/postgresql/data
    networks:
      - vending_network
    healthcheck:
      test: [ "CMD-SHELL", "pg_isready -U vending_user -d vending" ]
      interval: 10s
      timeout: 5s
      retries: 5

  backend:
    image: ghcr.io/aldervon-systems/vendingbackpack:latest
    container_name: vending_backpack_backend
    restart: unless-stopped
    ports:
      # FIX: Map Host Port 9101 -> Container Port 8080
      - "9101:8080"
    environment:
      DATABASE_URL: postgresql://vending_user:iq200196Qr51@postgres:5432/vending
      DEMO_MODE: "true"
      ENVIRONMENT: production
    depends_on:
      postgres:
        condition: service_healthy
    networks:
      - vending_network

  frontend:
    image: ghcr.io/aldervon-systems/vendingbackpack/frontend-flutter:latest
    container_name: vending_frontend
    restart: unless-stopped
    ports:
      - "80:80"
    networks:
      - vending_network
    depends_on:
      - backend

volumes:
  postgres_data:
    driver: local

networks:
  vending_network:
    driver: bridge
```
